# Access Model

## Purpose

Clarify the boundary between public lead-capture assets and the existing Contractor Circle Hub.

Do not assume the Hub is public.

The Hub remains gated.

The public side diagnoses pain and invites people into Contractor Circle.

The Hub side delivers implementation, full tools, call replays, assignments, tracks, and member wins.

## Access Rule

```text
Public gets diagnosis and lite tools.

Contractor Circle Hub gets implementation and full tools.
```

## Public Free Assets

These assets can live on public landing pages, email forms, Instagram link-in-bio, or other public surfaces.

Purpose:

- Capture leads.
- Diagnose pain.
- Segment the lead by biggest leak.
- Invite to Contractor Circle.

Assets:

- Contractor Profit Leak Audit.
- Change Order Cash Tracker lite.
- Landing pages.
- Thank-you pages.
- Email sequence.
- Instagram link-in-bio.

Public CTA:

```text
The audit shows where the leak is.

Contractor Circle is where we fix it.
```

## Public Preview Assets

These assets can show what exists inside Contractor Circle without giving away the full implementation environment.

Allowed previews:

- Blurred screenshots.
- Sample/fake company dashboards.
- Loom walkthrough clips.
- Tool teaser screenshots.
- Before/after examples.
- Public teardowns.
- Call clips.
- Member win summaries.
- "Inside Contractor Circle" posts.

Purpose:

- Make the gated environment tangible.
- Create desire for access.
- Prove the tools exist.
- Show the type of work done inside the Circle.

Do not publish:

- Full member workflows.
- Full tool templates.
- Full call replays.
- Member-only assignments.
- Full implementation tracks.
- Private member data.
- Private member discussions.

## Member-Only Hub Assets

These assets belong inside the Contractor Circle Hub.

Purpose:

- Paid implementation environment.
- Tool adoption.
- Member retention.
- Member wins.
- Operating system installation.

Assets:

- Start Here / Profit Leak Audit for members.
- Full Change Order Cash system.
- Full Estimating Profit system.
- Full Project Profit / IOR / OverWatch tools.
- Full AOS tools.
- Implementation tracks.
- Call replays.
- Assignments.
- Wins board.
- Member roadmap.
- Tool index.
- Call archive organized by problem.

## The Audit In Both Places

The Contractor Profit Leak Audit can exist in two forms:

1. **Public audit**
   - Used for lead capture.
   - Captures email.
   - Diagnoses the biggest leak.
   - Routes to email nurture and Contractor Circle invitation.
   - Gives a useful result, not the full implementation path.

2. **Hub audit**
   - Used for members.
   - Routes members into implementation tracks.
   - Connects to tools, assignments, calls, and wins board.
   - Supports the 30-day member reset and ongoing onboarding.

Same diagnostic logic.

Different access level.

## Existing Member Reset Flow

Location:

- Contractor Circle Hub.

Flow:

1. Member sees reset announcement.
2. Member takes the Hub version of the Contractor Profit Leak Audit.
3. Member is assigned to a track.
4. Member receives one tool assignment.
5. Member attends or watches the reset call.
6. Member posts next action.
7. Member reports one win.
8. Win becomes proof, call topic, or content seed.

Member tracks:

- Stop Free Work.
- Fix The Estimate.
- Control The Job.
- Get Out Of The Middle.

## New Member Onboarding Flow

Location:

- Contractor Circle Hub.

Flow:

1. New member enters Start Here.
2. New member takes the Hub Profit Leak Audit.
3. New member chooses or is assigned a first 30-day path.
4. Hub shows the assigned tool path.
5. Member watches the relevant call replay or attends the next implementation call.
6. Member completes one assignment.
7. Member posts the result or stuck point.
8. Member gets routed to the next action.

First 30-day paths:

- Stop Free Work: Change Order Cash system.
- Fix The Estimate: Estimating Profit system.
- Control The Job: Project Profit / IOR / OverWatch tools.
- Get Out Of The Middle: AOS tools, roles, scorecards, and rhythm.

- Teaching clips.
- Call clips.

Public language:


## What Should Stay Gated

Keep these inside the Hub:

- Full tools.
- Full workbooks.
- Full tracker systems.
- Full IOR / OverWatch workflows.
- Full AOS implementation tools.
- Call replays.
- Member assignments.
- Member discussions.
- Wins board.
- Member roadmap.
- Tool usage path.
- Internal member dashboards.

Gated language:

```text
Members get the full tool, the implementation path, the call replay, and the weekly assignment inside Contractor Circle Hub.
```

## Practical Rule For Future Build Work

When creating a new asset, decide which access level it belongs to before building:

1. Public free diagnostic.
2. Public preview.
3. Member-only Hub tool.
4. Premium bounded access.

If the asset tells someone what is wrong, it can usually be public.

If the asset gives them the full implementation system, it belongs in the Hub.
